# Processo XP + Scrum

## Práticas de XP adotadas

- Programação em pares
- Integração contínua
- Design simples
- Refatoração
- Testes frequentes

## Integração com Scrum

Scrum organiza o trabalho (sprints, cerimônias)  
XP define como desenvolver o código (qualidade técnica)

## Fluxo semanal

- Sprint Planning (segunda)
- Daily (todos os dias)
- Desenvolvimento contínuo
- Review (sexta)
- Retrospectiva (sexta)

## Cronograma da Sprint (2 semanas)

### Semana 1
- Planning (2h)
- Dailies (15min/dia)
- Desenvolvimento + XP

### Semana 2
- Continuação dev
- Review (1h)
- Retrospectiva (1h)

## Scrum vs Kanban

| Aspecto | Scrum | Kanban |
|--------|------|--------|
| Uso | Projetos com sprints | Fluxo contínuo |
| Estrutura | Iterações fixas | Contínuo |
| Mudanças | Dentro da sprint não | A qualquer momento |
| Métrica | Velocity | Lead time |

### Combinação
Usar Scrum para gestão e Kanban para visualização.
